# Security Policy

## Supported Versions
We generally support the currently deployed production branch. Older branches may not receive security updates unless explicitly noted.

## Reporting a Vulnerability
- Email: security@keralacoderscafe.in (preferred)
- Alternatively, open a **private** security advisory on GitHub (Security ▸ Advisories ▸ Report a vulnerability).

Please include:
- A clear description of the issue and impact
- Steps to reproduce or proof-of-concept
- Affected endpoints/URLs or components
- Any known mitigations

## Disclosure
We aim to acknowledge receipt within 72 hours and provide an initial assessment within 7 days. We prefer coordinated disclosure; please do not open public issues for security findings.

## Safe Harbor
We will not pursue legal action against good-faith security research that:
- Respects user privacy and does not disrupt production systems
- Avoids accessing, modifying, or deleting data that you do not own
- Gives us reasonable time to remediate before public disclosure
